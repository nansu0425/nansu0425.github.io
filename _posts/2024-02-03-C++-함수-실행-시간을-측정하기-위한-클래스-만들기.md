## 1. 개요
클래스 이름은 FunctionSpeedTest이다. 밀리초 단위의 함수 실행 시간을 표준 출력 스트림으로 출력한다. 테스트 대상 함수는 Input 폴더 파일들로부터 입력을 받고 출력을 Output 폴더에 파일로 저장한다.

> <a href='https://github.com/nansu0425/FunctionSpeedTest.git' target='_blank'>FunctionSpeedTest Github Repository</a>

## 2. 만든 계기
백준에서 C++ 입출력 연습 문제를 풀다가 어떤 입출력 함수가 빠른건지 궁금해졌다. 검색해보니 입출력 함수의 실행 시간이 잘 정리된 표가 있었다. 그 표를 보고 그냥 어떤 함수가 빠르구나 하고 넘어갈 수도 있었지만 내가 실행 시간을 측정해서 직접 확인해보고 싶었다. 실행 시간 측정 방법을 찾아보니 간단했다. 함수 실행 전 time point를 찍고 함수 종료 후 다시 time point를 찍어서 두 시간의 차를 구하는 것이다. 간단한 작업이지만 어떤 함수의 실행 시간이 궁금할 때마다 이 작업을 하기에는 매우 귀찮을 것 같았다. 만약 여러 경우에 대한 테스트를 하려고 하면 더욱 귀찮아진다. 이런 생각을 하다 결국 함수 실행 시간 측정을 편하게 할 수 있는 도구를 만들기로 했다.

## 3. 클래스 구조

### 3.1. 데이터 멤버

- **m_target_ptr**

    테스트할 함수를 가리키는 포인터다. 이 함수의 실행 시간을 측정한다.

- **m_count**

    함수를 테스트하는 횟수를 설정한다. 실행 시간을 구할 때 평균 시간을 계산한다.

### 3.2. 멤버 함수

- **FunctionSpeedTest**

    클래스의 생성자다. `m_target_ptr`과 `m_count`를 초기화한다. `m_count`에 인자가 없으면 1로 설정한다.

- **test**

    `m_target_ptr`이 가리키는 함수를 테스트한다. `time_ptr`은 실행 시간을 저장할 벡터의 포인터다. 테스트는 과정은 Input 폴더에 파일이 있는지 없는지로 방법이 나뉜다.

    - Input 폴더에 파일이 없는 경우

        1. `std::cout`을 Output 폴더의 output.txt 파일로 리다이렉트한다.
        2. `get_execution_time`으로 `m_target_ptr` 함수의 실행 시간을 얻는다.
        3. `std::cout`의 출력 스트림을 리다이렉트 하기 전으로 변경한다.
        4. `get_ms_str`으로 밀리초 단위 실행 시간 문자열을 얻은 후 출력한다.

    - Input 폴더에 파일이 있는 경우

        1. `std::cin`을 Input 폴더의 첫 번째 파일로 리다이렉트한다.
        2. `std::cout`을 입력 파일 이름과 같은 Output 폴더의 파일로 리다이렉트한다. 
        3. `get_execution_time`으로 `m_target_ptr` 함수의 실행 시간을 얻는다.
        4. `std::cout`, `std::cin`의 입출력 스트림을 리다이렉트 하기 전으로 변경한다.
        5. `get_ms_str`으로 밀리초 단위 실행 시간 문자열을 얻은 후 출력한다.
        6. Input 폴더에 다음 입력 파일이 있으면 `std::cin`을 리다이렉트한 후 2번부터 다시 진행한다. 다음 입력 파일이 없을 때까지 반복한다. 

- **get_execution_time**

    `m_target_ptr` 함수를 `m_count` 횟수만큼 반복하여 호출한 후 실행 시간의 평균을 반환한다. 반복할 때마다 마지막에 `ofs_ptr`, `ifs_ptr`로 파일 내의 입출력 위치를 처음으로 되돌린다. `time_ptr_ref`가 nullptr이 아니면 실행 시간 평균을 `time_ptr_ref`가 가리키는 벡터 끝에 추가한다. 

- **get_ms_str**

    실행 시간 데이터를 받으면 밀리초(ms) 단위의 문자열을 반환한다.

## 5. 느낀점

기초적인 C++ 문법을 익히고 진행해본 첫 C++ 프로젝트였다. 프로젝트라고 부르기에도 민망한 아주 작은 스케일이다. 3일 정도 걸린 것 같다. 하지만 단순히 연습하기 위한 목적이 아닌 필요에 의해 내가 사용할 도구를 직접 만들었다는 점은 뿌듯하다. 주로 입출력과 파일을 다루는 함수들을 이용했는데 아직 이 부분에 대해 이해가 부족함을 느꼈다. 필요한 기능을 먼저 구현하는 것을 목표로 했기에 예외처리나 안정성 부분은 별로 고려하지 못 했다. 이 부분에 대해서도 추가적인 공부가 필요해보인다. 그리고 C++ 코딩 스타일에 대해 공부해야겠다는 생각이 들었다. 코드를 짜면서 C++에 적합한 스타일인지 계속 의구심이 들었고 확신이 없었다. 객체지향의 특성을 적극 살리고 싶었지만 프로젝트의 크기가 작아서 그런지 그럴 여지가 별로 없었던 것 같다. 부족하다고 느낀 점이 많았지만 파일과 스트림에 대한 이해가 전보단 확실히 높아졌다는 느낌이 든다. 표준 스트림의 파일 리다이렉트, 파일을 접근하고 다루는 방법 등에 대해 배울 수 있었다.

## 6. 향후 계획
프로젝트를 진행하면서 만났던 문제들에 대해 분석한 글을 정리해서 포스팅할 예정이다. 그리고 하다보니 욕심이 생겨 프로젝트를 확장하고 싶은 생각이 들었다. 가장 마음에 안들었던 점은 `std::cin`, `std::cout`만 리다이렉트 했기 때문에 `printf`나 `scanf`를 사용하면 콘솔로 입출력이 이루어진다는 것이다. 특정 입출력 함수가 아니라 프로세스를 리다이렉트하는 방향으로 수정할 예정이다. 또한 단순히 함수의 실행 시간을 측정하는 것에 그치지 않고 코딩테스트의 테스트 케이스를 확인하는 것처럼 동작하도록 만들고 싶다. 입력으로 테스트 케이스에 대한 입력 파일, 출력 파일이 주어지면 각 테스트 케이스의 실행 시간과 실제 출력과 비교해 성공 여부를 출력하는 프로그램을 구상 중이다. 조금 더 욕심낸다면 서버 형태로 만들어볼 수도 있을 것 같다.
